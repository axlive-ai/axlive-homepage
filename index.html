FRAMEWORK

AI Agent는
행동으로 설계되어야 합니다

AX Live는 Agent가 언제, 무엇을, 어떻게 행동해야 하는지를 정의합니다.
화면이 아닌 행동을 설계하는 프레임워크입니다.

01 Principles

AI Agent의 행동은 다섯 가지 질문에서 시작됩니다.
누구에게, 언제, 무엇을, 어떻게, 왜 행동해야 하는지를 정의합니다.

WHO

누구에게

어떤 사용자인지에 맞춰 행동을 최적화

WHEN

언제

어떤 상황·조건에서 개입하고 시작할지

WHAT

무엇을

어떤 일을 하고, 하지 않아야 하는지

HOW

어떻게

어떤 순서와 톤, 근거로 응답하는지

WHY

왜

그 행동을 선택한 이유를 설명·통제

02 Components

행동은 여섯 가지 요소로 완성됩니다.
대화·상태·맥락·가드레일·의사결정·사람전환까지. 여섯 요소가 하나의 일관된 행동 체계를 만듭니다.

Conversation · 대화

어떤 순서와 톤으로 응답할지

State · 상태

지금 어떤 단계·모드에 있는지

Context · 맥락

무엇을 알고 무엇을 참조하는지

Guardrails · 가드레일

해서는 안 되는 것과 안전 경계

Decision Flow · 의사결정

조건에 따라 어떻게 분기·판단하는지

Human Handoff · 사람 전환

언제 사람에게 넘기고 승인받는지

SERVICE

금융 서비스를
Agent 기반으로 전환합니다

표준 업무프로세스부터 Agent Experience까지.
AX Live는 금융 업무를 AI Agent가 수행할 수 있는 서비스로 전환합니다.

01 Playbook

스크린 UI 중심의 업무를 대화 중심으로 전환하기 위해, Agent 업무프로세스의 표준을 정의합니다.

Agent 표준 업무프로세스

업무 담당자의 경험에 의존하던 업무를 Agent가 실행 가능한 업무 기준으로 전환합니다.

PROCESS
Analyze 현재 업무의 절차와 판단 기준, 예외 상황을 분석합니다.
↓
Define 업무 규칙과 판단 기준을 Agent 관점으로 정리합니다.
↓
Standardize 업무별 Playbook으로 체계화해 모든 Agent가 같은 기준으로 동작하게 합니다.
02 Agent Service Planning

고객의 성향과 목적에 따라 사용자와 운영자 모두에게 최적화된 Agent 서비스를 기획합니다.

Agent 서비스 기획

화면을 대화형으로 옮기는 것을 넘어, 여러 Agent가 하나의 서비스처럼 작동하는 구조를 만듭니다.

PROCESS
Organize 고객의 목적과 업무 특성, 사용자·운영자의 흐름을 정리합니다.
↓
Connect Welcome · Master · Sub Agent의 역할을 나누고 연결합니다.
↓
Orchestrate Agent Hub와 Dashboard로 전체 서비스를 하나로 운영합니다.
03 Service Concept

서비스의 장점과 핵심 고객의 니즈를 바탕으로 명확한 컨셉을 도출해 Agent 서비스에 적용합니다.

서비스 컨셉

기술보다 경험이 서비스를 가릅니다. 브랜드에 맞는 Agent의 태도와 대화 경험을 만듭니다.

PROCESS
Discover 기업의 철학과 브랜드, 핵심 고객의 특성을 파악합니다.
↓
Shape Agent의 말투와 행동, 대화의 방향을 잡습니다.
↓
Express 차별화된 AI UX로 브랜드 경험을 담아냅니다.
04 Agent Experience System

앞 단계의 업무 기준과 서비스 경험을 하나의 Agent Experience System으로 구성합니다.

Agent Experience System

앞선 세 단계의 기준을 실제 서비스에서 일관되게 작동하도록 하나의 체계로 묶습니다.

PROCESS
Systemize 정의된 대화·응답·상호작용 기준을 패턴으로 정리합니다.
↓
Integrate Conversation · Response · Interaction Pattern을 하나의 체계로 통합합니다.
↓
Operate Agent 서비스에서 그대로 작동하는 Agent Experience로 운영합니다.

INSIGHT

AI Agent 시대의 변화를
금융 서비스 관점에서 해석합니다

AX Live는 금융 서비스가 Agent 중심으로 어떻게 변화하는지 분석하고,
실제 프로젝트에 적용할 수 있는 인사이트를 제공합니다.

Agent 설계2026. 07. 08 · 6분 읽기

스크린을 옮기지 마세요. 업무를 다시 그리세요

기존 앱 화면을 그대로 대화로 옮기면 왜 편의성이 오히려 떨어지는가.
Agent 전환에서 가장 흔한 실패 패턴과, 업무를 처음부터 다시 그려야 하는 이유를 정리했습니다.
전체 Agent 설계 금융 AX 방법론 회사 소식
금융 AX2026. 07. 02

규제 산업에서 Agent를 검증한다는 것

은행 컴플라이언스가 진짜로 묻는 질문은 "잘 되나요"가 아니라 "벗어난 요청에 어떻게 반응하나요"입니다. Behavior Assurance가 필요한 이유.
방법론2026. 06. 24

Welcome · Master · Sub Agent, 왜 나누는가

하나의 거대한 Agent 대신 역할을 나누고 연계했을 때 사용자 경험과 운영이 어떻게 달라지는지, 실제 구성 원칙을 공유합니다.
Agent 설계2026. 06. 15

대화에도 상태(State)가 필요하다

지금 사용자가 어떤 단계에 있는지 Agent가 모르면 대화는 무너집니다. 행동 설계에서 State를 다루는 방법.
금융 AX2026. 06. 03

대고객 컨셉이 위닝 포인트가 되는 순간

자동화된 시스템과 획일적인 서비스 환경일수록, 차별화된 컨셉이 승부를 가릅니다. 컨셉을 시각화해 Agent에 적용한 사례.
